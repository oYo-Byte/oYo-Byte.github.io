学习PostgreSQL的查询机制

概述

在PostgreSQL中，查询语句经过以下5个子系统处理：

  1. Parser
    用于将文本式的SQL命令转换成解析树

  2. Analyzer/Analyser
    用于执行解析树的语义分析并生成查询树

  3. Rewriter
    重写器用于根据规则系统中已存在的规则转换查询树
  4. Planner
    规划器生成可以从查询树中最有效地执行的计划树
  5. Executor
    执行器通过按计划树创建的顺序访问表和索引来执行查询
